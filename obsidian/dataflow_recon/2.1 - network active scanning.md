#phase

run only when ASN/IP scope is explicit. on apex-domain-only SaaS BBPs, skip this entire phase

asnmap -> CIDR expansion
	`cat asns.txt | asnmap -silent > asnmap_ranges.txt`
	merge with manual ip_ranges.txt, dedupe

naabu (web ports only — noise control)
	`cat asnmap_ranges.txt | naabu -p 80,443,8080,8443,4443,8888,9000 -silent -o naabu_live_ports.txt`
	going broad (-p -) wastes time and triggers WAF alerts; web-only ports cover the BAC-relevant surface

diff to surface IPs with no DNS record
	`comm -23 <(awk -F: '{print $1}' naabu_live_ports.txt | sort -u) <(dig +short -f subdomains_resolved.txt | sort -u) > ip_no_dns.txt`
	the high-value output of this phase: shadow IT, forgotten staging, decommissioned-from-DNS-but-still-live infra. these IPs are invisible to other hunters running pure subdomain enum

service banners (optional, only on novel ports if naabu surfaces them)
	`naabu -l ip_no_dns.txt -nmap-cli 'nmap -sV -Pn'`
	in bug bounty, non-web services rarely chain — skip unless something exotic shows up

-- input:
	asns.txt, ip_ranges.txt
	subdomains_resolved.txt  (for diffing)

-- output:
	[[asnmap_ranges.txt]]
	[[naabu_live_ports.txt]]
	[[ip_no_dns.txt]]
